# Admin Workflows

Everything a Curve consultant does inside `/admin/*`. Ordered roughly by cadence.

## Daily

**Triage inbox → `/admin`**
Dashboard cards show orgs with unread messages, blocked tasks, warm sponsorship leads, and overdue compliance items. Click any card to jump into that org.

**My tasks → `/admin/my-tasks`**
Any `org_tasks.owner = 'curve'` row where `assignee = you`. Complete inline — the trigger will recompute revenue share if the task's completion writes to `org_revenue_entries`.

**Meetings inbox (Acquisitions) → `/admin/acquisitions/meetings`**
Fathom/Fireflies transcripts pushed via `fathom-webhook` land here. Click one → `TranscriptDetail` runs `process-meeting-transcript` to extract action items, decisions, and risks into `acquisition_meeting_action_items`.

## Weekly

**Set the weekly focus per org → `/admin/weekly-focus`**
Writes `org_weekly_focus`. This is what shows up at the top of the org's dashboard and drives the Monday digest (`send-weekly-rollup` → `send-weekly-focus-email`).

**Review the portfolio → `/admin/marketing/portfolio` and `/admin/health`**
`curve_portfolio_summary` view rolls up revenue share, plan-completion %, marketing engagement, and NPS across every org.

**Task tracker → `/admin/task-tracker`**
Bird's-eye view of every open `org_tasks` row across all orgs. Bulk reassign / re-due / snooze.

## When onboarding a new org

1. **Create org** — `AdminOrganizationsList` → New Org.
2. **Invite the primary user** — `/admin/invite` (sets `pending_invitations.role='org_user'`, marks that user as `is_primary=true` in the invitation).
3. **They complete intake** (`/intake`). `calc-metrics` edge function fires on submit and populates `derived_metrics`.
4. **Review draft plan** — `/admin/org/:orgId` → Plan tab. Draft tasks generated by `generate-plan-from-templates` sit there awaiting approval.
5. **Edit / trim / add tasks** in draft state.
6. **Approve plan** — the "Activate Plan" action opens `BaselineModal`. Enter baseline revenue and contract value. This writes `org_engagement_baselines`, `org_engagement_contracts`, and sets `organizations.plan_activated_at`.
7. **Release the first project** — Projects tab → New Project → pick tasks → set phases → Publish. Org can now see those tasks.

## Managing a project (post-approval)

`/admin/org/:orgId/projects/:projectId` — the dedicated project detail page.

- **Drag-and-drop tasks** across phases (uses `@dnd-kit`). On drop the client writes `phase` and `display_order` back to `org_tasks`.
- **Edit inline** — title, description, assignee, due date, phase.
- **Hard gate** — the `task_phase_is_unlocked` trigger blocks completion of a task if any earlier-phase task is incomplete. Admins are exempt.
- **Delete project** — leaves the tasks in place but sets `project_id = null` (unassign, don't destroy).

## Revenue share

`/admin/revenue-share` → per-org drill-down at `/admin/revenue-share/:orgId`.

- Log new revenue: click Add Revenue → writes `org_revenue_entries` → trigger recomputes `org_revenue_share_summary`.
- Sponsorship deals from the pipeline sync automatically via `sync_sponsorship_to_revenue`.
- The org sees the same numbers, but read-only.

## Communications

`/admin/communications` (allegiance-only) — a portfolio inbox of every DM thread across every org (`communication_threads` / `communication_messages`). Use this to answer org questions without leaving admin.

## Sales

- **Revenue Audits (leads)** — `/admin/revenue-audits`. Every public form submission.
- **Pipeline** — `/admin/pipeline`. Curve's own sales pipeline for prospects (not org-scoped).

## System health

- **`/admin/system/wiring-status`** — checks each critical edge function's last successful invocation, queue depth in pgmq, and unresolved errors in `admin_error_log`.
- **`/admin/health`** — org-facing health reports (plan progress, NPS, marketing engagement).
